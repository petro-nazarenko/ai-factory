# AI Factory — Аудит проекта

> Дата аудита: 2026-04-05  
> Версия: 0.2.0  
> Статус: Alpha — **не готов к production**

---

## 1. Архитектура и связность модулей

### 1.1 Жёсткая связность через bash-скрипт

| | |
|---|---|
| **Файл** | `run_pipeline.sh` |
| **Проблема** | Все шаги pipeline вызываются как отдельные процессы через bash. Нет возможности запустить один шаг без запуска всех предыдущих. Ошибка в любом шаге останавливает скрипт без rollback. |
| **Риск** | HIGH |
| **Оценка** | 4 ч |

**Решение:** реализовать `workspace/pipeline.py` с классом `Pipeline`, где каждый шаг — независимо переиспользуемый метод.

---

### 1.2 `workspace/connector.py` — нет валидации входного формата

| | |
|---|---|
| **Файл** | `workspace/connector.py` |
| **Проблема** | Connector читает `ideas.json` без проверки структуры. При неожиданном формате (например, `idea` → `null`) скрипт падает с `KeyError` без информативного сообщения. |
| **Риск** | MED |
| **Оценка** | 1 ч |

**Решение:** добавить Pydantic-модель `IdeaInput` для валидации при чтении.

---

### 1.3 `workspace/llm_router.py` — fallback не логирует причину переключения

| | |
|---|---|
| **Файл** | `workspace/llm_router.py` |
| **Проблема** | При переключении провайдера (Groq → Cerebras → Anthropic) причина fallback не попадает в `logs.txt`. В production невозможно диагностировать частоту исчерпания лимитов. |
| **Риск** | MED |
| **Оценка** | 1 ч |

---

## 2. API (ai-factory-api)

### 2.1 Нет rate limiting на `POST /run`

| | |
|---|---|
| **Файл** | `ai-factory-api/main.py` |
| **Проблема** | Endpoint `POST /run` запускает новый subprocess без ограничения количества параллельных запусков. Злоумышленник или ошибка клиента могут породить сотни процессов. |
| **Риск** | HIGH |
| **Оценка** | 2 ч |

**Решение:** хранить PID активных запусков в памяти, ограничить параллельные запуски (например, 3).

---

### 2.2 API-ключ хранится в одной переменной окружения без ротации

| | |
|---|---|
| **Файл** | `ai-factory-api/main.py` |
| **Проблема** | `API_KEY` — одна статичная строка. Нет поддержки нескольких ключей, нет механизма отзыва. |
| **Риск** | MED |
| **Оценка** | 2 ч |

---

### 2.3 `GET /runs/{run_id}` не ограничивает размер `validated/*.md`

| | |
|---|---|
| **Файл** | `ai-factory-api/main.py` |
| **Проблема** | Все `validated/*.md` файлы читаются целиком и возвращаются в одном JSON-ответе. При большом количестве идей ответ может превысить лимиты Railway. |
| **Риск** | LOW |
| **Оценка** | 1 ч |

---

## 3. Moneymaker (signal mining)

### 3.1 `jobboards.py` — нет обработки HTTP timeout

| | |
|---|---|
| **Файл** | `Moneymaker/src/signal_miner/jobboards.py` |
| **Проблема** | Запросы к HN Algolia API и RemoteOK идут без явного timeout. При зависшем соединении шаг mining блокирует весь pipeline indefinitely. |
| **Риск** | HIGH |
| **Оценка** | 1 ч |

**Решение:** добавить `timeout=httpx.Timeout(10.0)` для всех HTTP-вызовов.

---

### 3.2 `money_filter.py` — критерии фильтра хардкодированы

| | |
|---|---|
| **Файл** | `Moneymaker/src/money_filter.py` |
| **Проблема** | Порог score `>= 7.0` и список критериев зашиты в код. Невозможно изменить фильтрацию без правки файла. |
| **Риск** | LOW |
| **Оценка** | 2 ч |

**Решение:** вынести в конфиг (`Moneymaker/.env` или `config.py`).

---

## 4. AKF (ai-knowledge-filler)

### 4.1 `akf.py` — retry-логика не покрыта тестами

| | |
|---|---|
| **Файл** | `ai-knowledge-filler/akf.py` |
| **Проблема** | Согласно spec, при ошибке валидации выполняется до 2 retry. Этот путь не покрыт unit-тестами — ошибка в retry-логике не будет обнаружена до production. |
| **Риск** | MED |
| **Оценка** | 3 ч |

---

### 4.2 `system_prompt.md` — не версионируется отдельно

| | |
|---|---|
| **Файл** | `ai-knowledge-filler/akf/system_prompt.md` |
| **Проблема** | Изменения в system prompt меняют поведение всей валидации, но нет changelog для prompt-версий. Откат после регрессии требует ручного поиска по git. |
| **Риск** | LOW |
| **Оценка** | 2 ч |

## 5. Безопасность

### 5.1 `.env`-файлы могут попасть в репозиторий

| | |
|---|---|
| **Файл** | `.gitignore` |
| **Проблема** | `Moneymaker/.env` упомянут в README как место хранения API-ключей. Если `.gitignore` не покрывает все `**/. env` паттерны, ключи могут быть закоммичены. |
| **Риск** | CRITICAL |
| **Оценка** | 0.5 ч |

---

### 5.2 `run_pipeline.sh` — нет проверки integrity outputs

| | |
|---|---|
| **Файл** | `run_pipeline.sh` |
| **Проблема** | После каждого шага нет проверки, что output-файл создан и не пустой. Следующий шаг получает пустой input и создаёт misleading output. |
| **Риск** | MED |
| **Оценка** | 1 ч |

---
